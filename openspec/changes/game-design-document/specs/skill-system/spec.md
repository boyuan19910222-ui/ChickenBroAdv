## 新增需求

### 需求: 技能分类
系统应支持三类技能：主动技能、被动技能和团队技能。

#### 场景: 主动技能使用
- **当** 玩家使用主动技能时
- **那么** 技能效果生效，冷却计时器开始

#### 场景: 被动技能效果
- **当** 玩家学习被动技能时
- **那么** 技能效果自动应用于角色

#### 场景: 团队技能协作
- **当** 玩家使用团队技能时
- **那么** 技能为范围内的队友提供增益效果

### 需求: 技能学习和进度
系统应允许玩家学习新技能，并通过练习和等级提升来提升现有技能。

#### 场景: 从训练师学习技能
- **当** 玩家访问技能训练师时
- **那么** 显示玩家职业和等级可用的技能

#### 场景: 技能等级提升
- **当** 玩家重复使用技能时
- **那么** 技能经验增加，可能提升到下一等级

### 需求: 技能冷却和资源
系统应为技能使用实现冷却时间和资源消耗。

#### 场景: 技能冷却管理
- **当** 玩家使用有冷却的技能时
- **那么** 技能在冷却结束前不可用

#### 场景: 法力消耗扣除
- **当** 玩家使用消耗法力的技能时
- **那么** 从角色扣除相应的法力值

### 需求: 职业专属技能
系统应为每个职业提供独特的技能组合。

#### 场景: 职业技能限制
- **当** 玩家尝试学习技能时
- **那么** 系统验证该技能是否对玩家职业可用

#### 场景: 跨职业技能限制
- **当** 玩家更改职业专精时
- **那么** 某些技能可能变为不可用或被修改

### 需求: 技能组合和增效
系统应支持技能组合，创造增强效果。

#### 场景: 技能连击执行
- **当** 玩家按顺序使用兼容技能时
- **那么** 触发额外的连击效果

#### 场景: 增效加成计算
- **当** 多个增效技能同时激活时
- **那么** 计算并应用增强效果
