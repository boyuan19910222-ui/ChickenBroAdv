## 新增需求

### 需求: 职业天赋树
系统应为9个职业中的每一个提供3个不同的天赋树。

#### 场景: 天赋树可用性
- **当** 玩家创建任意职业的角色时
- **那么** 该职业特有的3个天赋树变为可用

#### 场景: 天赋树专精
- **当** 玩家在某个天赋树中投入点数时
- **那么** 角色获得与该天赋树专注方向相关的专精加成

### 需求: 天赋点数分配
系统应在角色升级时奖励天赋点数，允许投资到天赋树中。

#### 场景: 天赋点数获得
- **当** 角色升级时
- **那么** 角色获得可消费的天赋点数

#### 场景: 天赋点数消费
- **当** 玩家消费天赋点数时
- **那么** 选择的天赋变为激活状态并提供其效果

### 需求: 天赋前置条件
系统应强制执行天赋前置条件，要求学习某些天赋后才能学习其他天赋。

#### 场景: 前置天赋检查
- **当** 玩家尝试学习某个天赋时
- **那么** 系统验证所有前置天赋是否已经学习

#### 场景: 天赋层级进度
- **当** 玩家在某个天赋树中投入足够的点数时
- **那么** 该天赋树中更高层级的天赋变为可用

### 需求: 天赋重置
系统应允许玩家重置并重新分配他们的天赋点数。

#### 场景: 天赋重置操作
- **当** 玩家使用天赋重置功能时
- **那么** 所有天赋点数被退还，可以重新分配

#### 场景: 重置费用递增
- **当** 玩家多次重置天赋时
- **那么** 后续重置的费用增加

### 需求: 天赋效果
系统应将天赋效果应用于角色的技能、属性和战斗表现。

#### 场景: 被动天赋效果
- **当** 玩家学习被动天赋时
- **那么** 天赋效果自动应用于角色

#### 场景: 主动天赋技能
- **当** 玩家学习主动天赋时
- **那么** 新技能变为可在战斗中使用
