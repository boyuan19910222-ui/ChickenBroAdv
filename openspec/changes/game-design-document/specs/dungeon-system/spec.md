## 新增需求

### 需求: 副本类型和难度
系统应提供多种副本类型，具有不同的难度等级。

#### 场景: 难度等级选择
- **当** 玩家进入副本时
- **那么** 系统根据玩家等级展示可用的难度等级

#### 场景: 难度奖励缩放
- **当** 玩家完成更高难度的副本时
- **那么** 奖励根据难度增加适当缩放

### 需求: 队伍组建
系统应支持为副本挑战组建队伍。

#### 场景: 队伍创建
- **当** 玩家为副本创建队伍时
- **那么** 系统允许邀请其他玩家加入

#### 场景: 职责平衡检查
- **当** 队伍进入副本时
- **那么** 系统推荐平衡的队伍配置（坦克、输出、治疗）

### 需求: 副本进度
系统应追踪副本进度，包括多个遭遇战和Boss战。

#### 场景: 遭遇战完成
- **当** 队伍击败副本中的遭遇战时
- **那么** 下一个遭遇战或区域变为可访问

#### 场景: Boss战机制
- **当** 队伍到达副本Boss时
- **那么** 独特的Boss机制和技能被激活

### 需求: 副本奖励
系统应为副本完成提供丰厚奖励，包括稀有装备和材料。

#### 场景: 完成奖励
- **当** 队伍成功完成副本时
- **那么** 所有队员获得经验和战利品奖励

#### 场景: 首次通关奖励
- **当** 玩家首次完成副本时
- **那么** 获得额外的奖励加成

### 需求: 副本重置机制
系统应实现副本重置计时器和锁定期限。

#### 场景: 每日副本重置
- **当** 每日重置发生时
- **那么** 已完成的副本重新变为可挑战状态

#### 场景: 锁定期限执行
- **当** 玩家完成某些副本时
- **那么** 临时锁定阻止立即重新进入
