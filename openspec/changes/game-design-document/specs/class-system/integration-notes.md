# 职业系统集成说明

## 与现有系统的集成关系

### 与角色系统 (character-system) 的集成
- **职业选择**：扩展了现有的9职业选择系统，提供详细的职业定义
- **职责分配**：与现有的坦克/DPS/治疗职责系统完全兼容
- **属性分配**：每个职业的属性需求与 attribute-system 中的属性影响机制对应

### 与技能系统 (skill-system) 的集成
- **职业技能限制**：每个职业的专属技能与现有技能分类系统兼容
- **技能树结构**：职业技能分支与现有的主动/被动/团队技能分类对应
- **跨职业限制**：确保职业专属技能的独占性

### 与天赋系统 (talent-system) 的集成
- **天赋树分支**：每个职业的3个天赋树与现有天赋系统框架一致
- **专精方向**：天赋专精支持不同的游戏风格和角色定位
- **天赋点分配**：遵循现有的天赋点获得和分配机制

### 与装备系统 (equipment-system) 的集成
- **护甲类型限制**：
  - 布甲：牧师、法师、术士
  - 皮甲：盗贼、德鲁伊（+布甲）
  - 锁甲：猎人、萨满祭司（+布甲+皮甲）
  - 板甲：战士、圣骑士（+布甲+皮甲+锁甲）
- **武器类型限制**：每个职业都有特定的可用武器列表
- **装备属性需求**：与各职业的主要属性需求对应

### 与战斗系统 (combat-system) 的集成
- **伤害计算**：职业属性加成与战斗伤害计算公式集成
- **资源系统**：不同职业使用不同的资源（怒气、法力、能量）
- **特殊机制**：职业独有机制（图腾、宠物、变形等）与战斗流程集成

### 与属性系统 (attribute-system) 的集成
- **主属性依赖**：
  - 力量：战士、圣骑士（惩戒）
  - 敏捷：猎人、盗贼
  - 智力：牧师、法师、术士、萨满祭司（元素/恢复）、德鲁伊（平衡/恢复）
  - 耐力：所有坦克职业和生存需求
  - 精神：所有施法职业的法力恢复
- **属性影响**：职业专精影响属性转换效率

## 特殊系统说明

### 职业独有机制
1. **猎人宠物系统**：需要额外的宠物管理和AI系统
2. **术士恶魔召唤**：需要恶魔类型数据库和召唤机制
3. **萨满图腾系统**：需要图腾放置和效果范围计算
4. **德鲁伊变形系统**：需要形态切换和技能限制机制
5. **盗贼潜行系统**：需要隐身检测和偷袭机制
6. **牧师种族法术**：需要种族-职业组合的特殊技能

### 平衡性考虑
- **角色定位明确**：每个职业都有其独特的优势和局限性
- **团队协作**：不同职业在团队中承担不同但同等重要的角色
- **多样化玩法**：同一职业的不同天赋专精支持不同的游戏风格
- **装备需求差异化**：避免职业间的装备竞争过于激烈

## 实现优先级建议

### 第一阶段：核心职业机制
1. 基础职业选择和属性分配
2. 职业装备限制系统
3. 基础技能和天赋框架

### 第二阶段：职业特色功能
1. 资源系统（怒气、能量、法力）
2. 基础职业技能实现
3. 简单的职业特殊机制

### 第三阶段：高级职业系统
1. 复杂的职业特殊机制（宠物、图腾、变形等）
2. 高级天赋和技能效果
3. 职业平衡性调优
