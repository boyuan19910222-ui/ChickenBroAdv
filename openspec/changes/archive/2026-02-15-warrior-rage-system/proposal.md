# 战士怒气系统

## 概述

实现通用资源系统框架，以战士怒气为首个实例。将现有的 mana-only 系统改造为支持多种资源类型（怒气、法力、能量）的通用架构。

## 背景

当前游戏中所有职业都使用 `mana` 作为技能资源，但这不符合经典 RPG 设计：
- 战士应该使用 **怒气**（通过战斗产生）
- 盗贼应该使用 **能量**（自动恢复）
- 施法职业使用 **法力**（静态池）

## 目标

1. 建立通用资源系统框架，支持不同资源类型的独特机制
2. 实现战士怒气系统作为首个非法力资源
3. 保持向后兼容，法师等职业继续使用法力

## 范围

### 包含
- 通用资源数据结构设计
- 战士怒气产生机制（被击中、普攻、暴击）
- 战士技能消耗怒气
- 脱战后怒气衰减
- UI 资源条适配显示
- 天赋修改资源上限的接口预留

### 不包含
- 盗贼能量系统（后续单独实现）
- 其他职业资源改造
- 装备影响资源的系统

## 成功标准

- [ ] 战士创建时使用怒气而非法力
- [ ] 战斗开始时怒气为 0
- [ ] 被击中获得 5 点怒气
- [ ] 普通攻击获得 8 点怒气（暴击 ×1.5）
- [ ] 技能正确消耗怒气
- [ ] 怒气上限默认 100
- [ ] 脱战 3 秒后开始衰减（每秒 -2）
- [ ] UI 正确显示怒气条（红色）
